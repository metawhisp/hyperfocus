## Project

Hyperfocus is a cinematic focus timer for macOS, built for people with ADHD and anyone who struggles to start a task and stay in it. A small glass **Focus Orb** floats on screen; the user clicks it, enters one **Mission** and a duration, and hits *Enter Hyperfocus*. A fullscreen **countdown** darkens the screen (`ENTER HYPERFOCUS MODE` → `3` → `2` → `1` → `FOCUS`), then a **green Aura Frame** glows along the screen edges while the timer runs. The Mac camera performs a local-only **presence check** (Vision face detection — nothing recorded, nothing uploaded): face missing 7 s → yellow warning; 15 s → red **away** state with paused timer, brown-noise alarm, and a "Session paused" card; 3 s of renewed presence → recovery back to green. When the timer reaches zero: "Mission complete" screen, session stats saved to a local JSON file. Native Swift/SwiftUI/AppKit, zero third-party dependencies, zero network code.

## Golden rules (digest — `CLAUDE.md` is the full, governing text)

1. **Ralph loop.** One checklist item per iteration: reread `CLAUDE.md` + `PROGRESS.md` → take the next unchecked item in `specs/05-implementation-plan.md` → implement → test → check it off → commit → repeat.
2. **TDD for logic.** `SessionReducer`, `SessionTimer` accounting, `SessionStore`, `OrbPositionStore` are pure and testable — failing test first, then code. Every transition in canon §4 gets a unit test.
3. **Surgical edits.** Touch only what the current item needs. No drive-by refactors, no "improving" neighboring code.
4. **No invented APIs, paths, or names.** Everything is specified — check `specs/00-canon.md` before writing a type, key, string, or file. If it's not specified, stop and check the specs; don't guess.
5. **Never rename canon identifiers.** Enum cases (`idle…exited`, `done/partial/notDone/exited`, `calm/strict/cinematic`), file names in canon §2, settings keys (`hf.*`, canon §8), copy strings (canon §9) are locked.
6. **Update `PROGRESS.md` every iteration** — current phase, what was closed, next step, known issues.
7. **Small atomic commits.** One commit = one closed checklist item / one green test. Clear message.
8. **Never commit** `Hyperfocus.xcodeproj` (generated by XcodeGen, git-ignored), secrets, or `DerivedData`.
9. **Blocked after 2–3 attempts on one item?** Stop. Write what you tried and where it fails into `PROGRESS.md` (Known issues) and end the iteration — don't grind blindly and don't silently skip.
10. **`// IMPLEMENT — see plan Phase N` markers in skeleton files are the Phase-0 deliverable, not TODO-stub violations** — remove each marker only when its plan item is closed.

## Build, run, test

Requirements: macOS 15+, Xcode 16+, XcodeGen.

```bash
brew install xcodegen        # once
xcodegen generate            # regenerates Hyperfocus.xcodeproj from project.yml
xcodebuild -project Hyperfocus.xcodeproj -scheme Hyperfocus -configuration Debug build
xcodebuild -project Hyperfocus.xcodeproj -scheme Hyperfocus test
open Hyperfocus.xcodeproj    # or run from Xcode
```

`project.yml` is the source of truth for the project; after changing it, rerun `xcodegen generate`. Swift 5 language mode, deployment target macOS 15.0, Apple frameworks only (canon §1).

## Privacy invariants — must never break

Enforced in code and in review (canon §6, §9, §11):

- **No recording, no saving, no uploading of camera frames.** No `AVCaptureMovieFileOutput`, no `AVAssetWriter`, no writing of `CVPixelBuffer`/`CGImage` anywhere.
- **No network code in the entire app.** No network entitlements.
- **Camera is off outside sessions.** The capture session is fully torn down (`stopRunning`, inputs/outputs removed) on the `stopCamera` effect and whenever a session ends.
- No identity recognition, no emotion detection — presence only (`VNDetectFaceRectanglesRequest`).
- No-camera fallback sessions must always remain available.

Any change that would touch these is out of scope by definition — stop and record it in `PROGRESS.md` instead.
